setPointerCapture

This test requires either Touch Events or Pointer Events, and illustrates the difference between the implicit and explicit capture in those event models.

For both buttons, pointermove and touchmove events fired on the element update the X/Y coordinates. The difference shows when you tap/click on the buttons and then move your finger/stylus/mouse outside of the element: with Touch Events and their intrinsic capture model expressed through targetTouches, the coordinates are still tracked, while by default Pointer Events cease to fire as soon as the pointer leaves the boundaries of the element, unless you use setPointerCapture.

First, a button without setPointerCapture.

The following button includes extra code to explicitly setPointerCapture on pointerdown. Pointer events are now fired even after pointer leaves the element, as long as the pointer is still down.
